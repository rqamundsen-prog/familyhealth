## 1. 品牌命名结论（已定稿）

- 中文主名：**我的情绪**
- 英文主名：**Within**
- 组合写法：**我的情绪 / Within**（对外）或 **我的情绪（Within）**（产品内）

### 1.1 为什么不是“家健”

“家健”偏系统名/功能名，效率导向强，但温度与私密感弱。

### 1.2 为什么是“我的情绪 / Within”

- **我的情绪**：强调自我关注、日常可进入、表达直观。
- **Within**：强调内在空间、边界感、私密性，符合心理照护语境。
- 中英组合后，兼顾：
  - 中文可理解性（降低首次认知成本）
  - 英文品牌气质（提升辨识度）

## 2. 核心产品哲学

> 这不是一个“管理情绪”的工具，而是一个“照护自己”的空间。

### 2.1 五个关键词

1. **私密（Private by default）**
2. **关怀（Warmth over control）**
3. **行动（Small but actionable）**
4. **连续（Tiny daily rhythm）**
5. **复盘（Pattern over panic）**

### 2.2 对应的设计原则

- 少诊断，多陪伴。
- 少命令，多邀请。
- 少宏大承诺，多可执行小步。
- 少“你应该”，多“你可以先试试”。

---

## 3. 产品定位与边界

### 3.1 这是啥

- 面向普通用户的日常情绪照护产品。
- 帮用户完成：记录 → 理解 → 小行动 → 周复盘。

### 3.2 这不是啥

- 不是临床诊断工具。
- 不是替代专业心理咨询或精神科服务。
- 不追求“内容堆叠型”心理课程平台路线。

### 3.3 风险边界

- 高风险状态必须明确提示线下求助路径。
- 产品提供支持，但不越位承诺“治疗效果”。

### 3.4 原子单位决策（2026-03-16）

- **原子单位：个人（而非家庭）**
- 家庭作为二级协作容器（可选开启），不是首屏必选前置。

决策理由：
1. 与“我的情绪 / Within”第一人称私密语义一致。
2. 降低首次使用门槛，用户在脆弱时刻可直接开始。
3. 更符合“默认仅自己可见”的隐私承诺。
4. 增长路径更自然：先个人见价值，再邀请家人协作。

## 4. 体验结构（信息架构建议）

建议主流程固定为四段：

1. **记录此刻**（30秒完成）
2. **理解状态**（命名情绪 + 触发因素）
3. **即时照护**（3–10分钟微行动）
4. **周期复盘**（7天趋势 + 下周一件事）

### 4.1 首页应该传递的感受

- “我被接住了”
- “我不需要一次解决全部问题”
- “我现在就能做一点有用的事”

---

## 5. 文案语气规范（Design Tone）

### 5.1 推荐词汇

- 看见、理解、照护、陪你、此刻、慢一点、先做一步

### 5.2 避免词汇

- 管控、纠正、必须、你应该、失败、异常

### 5.3 一句话品牌语（当前推荐）

**把情绪，好好放在心里。**

备选：
- 看见情绪，照护自己。
- 你的内在空间，值得被温柔对待。

---

## 6. 隐私哲学（Within 的兑现机制）

“Within”不是装饰词，必须有产品承诺：

1. 默认仅自己可见（首次进入明确告知）
2. 隐私设置入口常驻可达
3. 支持设备级上锁（Face ID/指纹/密码）
4. 导出与删除数据路径清晰
5. 任何分享动作均二次确认
